# Getting Started

## Prerequisites

- Node.js (version compatible with Nuxt 4)
- **pnpm** — this repo pins `packageManager: pnpm@11.12.0`. Don't use npm/yarn; the lockfile is pnpm-specific and mixing package managers will cause drift.

## Install

```bash
pnpm install
```

`postinstall` runs `simple-git-hooks` (sets up the commit-lint git hook) and `nuxt prepare` automatically — no extra step needed.

## Environment

```bash
cp config/example.env config/.env
```

The dev server reads env vars from `config/.env`, not a root `.env` file. This file is gitignored — the app **will not start** without it, so this step is not optional.

## Run

```bash
pnpm dev        # dev server
pnpm build      # production build
pnpm preview    # preview a production build locally
pnpm generate   # static build
```

## What you get on first run

- `/login` — a working login **form** (validation, loading state, redirect) that does **not**
  call a real API. It fabricates a user object client-side and stores it via `useAuth()`. See
  [authentication.md](./2.authentication.md).
- `/` and `/components/*` — a dashboard home page plus a set of reference pages, one per
  Nuxt UI component pattern (table, select, datepicker, upload, modal, loading states). Treat
  `pages/components/*` as a living style guide, not real product pages — copy patterns from
  them rather than building on top of them.

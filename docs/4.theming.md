# Theming

Colors and component styling are handled in three separate layers. Knowing which layer to
touch for a given change matters ‚Äî they don't overlap.

## Layer 1 ‚Äî Brand colors (`config/colors.ts`)

Base palette, defined once as hex values:

```ts
export const colors = {
  primary: '#3e58d2',
  secondary: '#496aeb',
  info: '#5793eb',
  success: '#03c948',
  warning: '#ffbb00',
  danger: '#fe1616'
};
```

A Nuxt build hook (`build:before`) runs `createAssetColors()`, which expands each hex color
into a full OKLCH shade ramp (50‚Äì950, via the `theme-colors` package) and writes the result to
`app/assets/css/colors.css` as CSS custom properties (`--color-primary-500`, etc.).

**`app/assets/css/colors.css` is generated output** ‚Äî it's marked "don't modify this file" at
the top. To change a brand color, edit `config/colors.ts` and rebuild; don't hand-edit the CSS.

## Layer 2 ‚Äî Runtime accent override (`app.vue`)

Independent of the build-time palette, users can pick a custom accent color at runtime.
`app.vue` reads a `__themecolor` cookie and, if it's a valid hex color, overrides
`--color-primary` and `--color-primary-*` directly on `document.documentElement` using the
same `theme-colors` + `chroma-js` logic ‚Äî but client-side, no rebuild.

This means **primary color has two possible sources**: the build-time default from
`config/colors.ts`, overridden per-user at runtime if the `__themecolor` cookie is set. Other
palette colors (`secondary`, `info`, `success`, `warning`, `danger`) are **not** subject to this
runtime override ‚Äî only `primary` is user-customizable today.

## Layer 3 ‚Äî Component-level styling (`app.config.ts` / `app/theme/`)

This layer is about shape/variant/slot overrides, not raw color values:

- `app.config.ts` ‚Äî Nuxt UI's app-level config: default color per semantic role, and slot
  overrides for components used in only one or two places.
- `app/theme/*.ts` ‚Äî larger, reusable overrides for specific components (datepicker, modal,
  sidebar, multi-select today) that would clutter `app.config.ts` or need to be shared across
  multiple usages of the same component.

## Quick reference

| I want to... | Edit |
|---|---|
| Change the brand's default primary/secondary/etc. color | `config/colors.ts`, then rebuild |
| Let users pick their own accent color | Already works ‚Äî see `app.vue`'s `__themecolor` cookie logic |
| Change how a specific Nuxt UI component looks everywhere (e.g. all modals) | `app/theme/` |
| Tweak a one-off component instance's slots/variant | `:ui="{ ... }"` prop on that component, or `app.config.ts` if it should apply app-wide |
| Add a brand-new theme role (not primary/secondary/info/success/warning/danger) | `config/colors.ts` ‚Äî add the key, `createAssetColors()` picks it up automatically |
